## 一、模块划分原则

### 1.1 核心原则

| 原则 | 说明 | 应用方式 |
|------|------|----------|
| **单一职责** | 每个模块只负责一个明确的功能领域 | 按技术层次划分：core、data、web、security |
| **高内聚低耦合** | 模块内部相关度高，模块间依赖少 | 通过接口隔离，依赖抽象 |
| **可复用性** | 通用功能抽取为独立模块 | 工具类、通用配置独立成模块 |
| **可维护性** | 模块边界清晰，便于维护 | 合理的包结构和命名规范 |
| **可扩展性** | 便于未来功能扩展 | 预留扩展点，支持插件化 |

## 五、模块间通信规范

### 5.1 依赖方向

```
应用层 ──────────────────────────────► 可以依赖
领域层 ──────────────────────────────► 基础设施层

```

| 调用方式 | 场景 | 技术 |
|----------|------|------|
| Feign Client | 同步调用 | OpenFeign |

### 8.1 模块创建模板

#### 8.1.1 Library 模块 pom.xml 模板

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <artifactId>daas-{父模块}</artifactId>
        <groupId>cn.healthcaredaas</groupId>
        <version>${revision}</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <artifactId>daas-{模块名}</artifactId>
    <name>DaaS : {模块名}</name>
    <description>{模块描述}</description>

    <dependencies>
        <!-- 依赖声明 -->
    </dependencies>
</project>
```

#### 8.1.2 Application 模块 pom.xml 模板

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

        <artifactId>daas-cloud</artifactId>

        <version>${revision}</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <artifactId>cloud-svc-{服务名}</artifactId>
    <name>DaaS Cloud : {服务名} Service</name>
    <description>{服务描述}</description>

    <dependencies>
        <!-- 依赖声明 -->
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```
