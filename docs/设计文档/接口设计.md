## 一、接口设计规范

### 1.1 RESTful API设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **资源命名** | 使用名词复数形式 | `/api/v1/datasources` |
| **HTTP方法** | 使用标准HTTP方法 | GET/POST/PUT/DELETE/PATCH |
| **版本控制** | URL中包含版本号 | `/api/v1/...` |
| **状态码** | 使用标准HTTP状态码 | 200/201/400/401/403/404/500 |
| **过滤排序** | 使用查询参数 | `?page=1&size=10&sort=createTime,desc` |

### 1.2 URL规范

```
/api/{version}/{resource}/{action}
```

**标准操作映射**:

| 操作 | HTTP方法 | URL | 说明 |
|------|----------|-----|------|
| 列表查询 | GET | `/api/v1/datasources` | 支持分页、过滤、排序 |
| 详情查询 | GET | `/api/v1/datasources/{id}` | 根据ID查询 |
| 创建 | POST | `/api/v1/datasources` | 创建资源 |
| 更新 | PUT | `/api/v1/datasources/{id}` | 全量更新 |
| 部分更新 | PATCH | `/api/v1/datasources/{id}` | 部分更新 |
| 删除 | DELETE | `/api/v1/datasources/{id}` | 删除资源 |
| 批量删除 | DELETE | `/api/v1/datasources/batch` | 批量删除 |
| 特殊操作 | POST | `/api/v1/datasources/{id}/test` | 自定义操作 |

### 1.3 请求参数规范

**分页参数**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码，从1开始 |
| size | int | 否 | 10 | 每页大小，最大100 |
| sort | string | 否 | - | 排序字段，如 `createTime,desc` |

**查询参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 否 | 关键字搜索 |
| status | string | 否 | 状态过滤 |
| startTime | datetime | 否 | 开始时间 |
| endTime | datetime | 否 | 结束时间 |

## 二、统一返回格式

### 2.1 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": { },
  "timestamp": 1648608000000,
  "requestId": "req_xxx"
}
```

### 2.2 分页响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [ ],
    "page": 1,
    "size": 10,
    "total": 100,
    "pages": 10
  },
  "timestamp": 1648608000000,
  "requestId": "req_xxx"
}
```

### 2.3 错误响应

```json
{
  "code": 40001,
  "message": "参数错误：数据源编码不能为空",
  "data": null,
  "timestamp": 1648608000000,
  "requestId": "req_xxx"
}
```

### 2.4 返回对象定义

```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    private Long timestamp;
    private String requestId;

    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }

    public static <T> Result<T> error(int code, String message) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMessage(message);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
}
```

### 2.5 分页对象定义

```java
@Data
public class PageResult<T> {
    private List<T> list;
    private Long page;
    private Long size;
    private Long total;
    private Long pages;
}
```

## 三、异常码定义

### 3.1 异常码规范

格式：`{类别码}{具体码}`

| 类别 | 范围 | 说明 |
|------|------|------|
| 系统级 | 10001-10999 | 系统通用错误 |
| 认证授权 | 20001-20999 | 登录、权限相关 |
| 参数校验 | 30001-30999 | 请求参数错误 |
| 业务错误 | 40001-49999 | 各业务模块错误 |
| 数据源 | 41001-41999 | 数据源相关错误 |
| 数据集成 | 42001-42999 | 数据集成相关错误 |
| 数据标准 | 43001-43999 | 数据标准相关错误 |
| 主数据 | 44001-44999 | 主数据相关错误 |
| 元数据 | 45001-45999 | 元数据相关错误 |
| 数据质量 | 46001-46999 | 数据质量相关错误 |
| 数据资产 | 47001-47999 | 数据资产相关错误 |
| 数据安全 | 48001-48999 | 数据安全相关错误 |
| HIE集成 | 49001-49999 | HIE集成相关错误 |
| 第三方服务 | 50001-59999 | 外部服务错误 |

### 3.2 通用异常码

| 码值 | 说明 |
|------|------|
| 200 | 成功 |
| 500 | 系统内部错误 |
| 10001 | 服务不可用 |
| 10002 | 远程服务调用失败 |
| 10003 | 限流触发 |
| 10004 | 服务降级 |
| 20001 | 未登录或登录已过期 |
| 20002 | 无访问权限 |
| 20003 | 认证失败 |
| 30001 | 参数错误 |
| 30002 | 参数缺失 |
| 30003 | 参数格式错误 |
| 30004 | 参数超出范围 |
| 40001 | 资源不存在 |
| 40002 | 资源已存在 |
| 40003 | 资源被占用 |
| 40004 | 资源状态错误 |
