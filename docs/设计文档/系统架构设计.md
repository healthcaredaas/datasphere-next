## 一、架构概述

### 1.1 设计原则

| 原则 | 描述 |
|------|------|
| 微服务架构 | 采用微服务架构，服务独立部署、独立扩展 |
| 领域驱动设计 | 按业务领域划分服务边界，实现高内聚低耦合 |
| 开放封闭 | 对扩展开放，对修改封闭，通过SPI机制扩展 |
| 单一职责 | 每个服务专注于单一业务领域 |
| 接口隔离 | 服务间通过明确定义的接口通信 |

### 1.2 技术选型

| 层次 | 技术组件 | 版本 | 选型理由 |
|------|----------|------|----------|
| 基础框架 | Spring Boot | 3.5.11 | 最新稳定版，性能优化，虚拟线程支持 |
| 微服务框架 | Spring Cloud | 2025.0.0 | 业界标准，生态完善 |
| 微服务治理 | Spring Cloud Alibaba | 2025.0.0 | 国内主流，组件丰富 |
| 服务注册/配置 | Nacos | 3.0.x | 支持服务注册和配置中心 |
| RPC框架 | Apache Dubbo | 3.3.x | 高性能RPC，支持多种协议 |
| 消息队列 | Apache RocketMQ | 5.3.x | 高可靠消息中间件 |
| 数据库 | MySQL | 8.0+ | 主流关系型数据库 |
| ORM框架 | MyBatis-Plus | 3.5.x | MyBatis增强，简化开发 |
| 数据集成 | Apache SeaTunnel | 2.3.x | 高性能数据集成引擎 |
| 任务调度 | PowerJob | 4.x | 分布式任务调度 |
| 缓存 | Redis | 7.x | 高性能缓存 |
| 数据库迁移 | Flyway | 10.x | 数据库版本管理 |

## 五、数据架构

### 5.1 数据库设计原则

| 原则 | 描述 |
|------|------|
| 命名规范 | 表名使用小写下划线，如 `datasource_info` |
| 主键设计 | 使用雪花算法生成分布式ID |
| 字段规范 | 必须包含 `id`, `create_time`, `update_time` |
| 索引设计 | 根据查询模式建立合适索引 |
| 版本管理 | 使用Flyway管理数据库变更 |

### 5.2 数据库分库策略

| 数据库 | 包含表 | 说明 |
|--------|--------|------|
| datasphere_base | 数据源、系统配置 | 基础数据库 |
| datasphere_integration | 数据集成相关表 | 集成数据库 |
| datasphere_standard | 数据标准相关表 | 标准数据库 |
| datasphere_master | 主数据相关表 | 主数据库 |
| datasphere_quality | 数据质量相关表 | 质量数据库 |
| datasphere_asset | 数据资产相关表 | 资产数据库 |
| datasphere_security | 数据安全相关表 | 安全数据库 |
| datasphere_hie | HIE相关表 | HIE数据库 |

### 5.3 缓存设计

#### 5.3.1 缓存策略

| 数据类型 | 缓存策略 | 过期时间 |
|----------|----------|----------|
| 数据源信息 | 读缓存 | 30分钟 |
| 字典数据 | 读缓存 | 1小时 |
| 用户权限 | 读缓存 | 30分钟 |
| 会话信息 | 读缓存 | 24小时 |
| 热点数据 | 读写缓存 | 5分钟 |

#### 5.3.2 缓存Key设计

```
# 数据源
datasphere:datasource:{id}

# 字典
datasphere:dict:{dictCode}

# 用户权限
datasphere:user:permission:{userId}

# 会话
datasphere:session:{sessionId}
```
