## 1. 概述

### 1.1 功能定位

数据质量模块是 DataSphere-Next 平台的核心组件，提供数据质量规则定义、检测执行、报告生成、问题跟踪的完整闭环管理能力。

### 1.2 核心能力

| 能力域 | 功能说明 |
|--------|----------|
| 规则管理 | 支持完整性、唯一性、格式、值域、自定义规则的定义和管理 |
| 模板机制 | 预置常用规则模板，支持快速创建规则 |
| 检测执行 | 支持即时检测和定时调度检测 |
| 质量报告 | 生成数据质量评分、问题分布统计报告 |
| 问题跟踪 | 质量问题记录、指派、处理闭环 |

### 1.3 技术栈

- **框架**: Spring Boot 3.5.x, MyBatis-Plus
- **数据库**: MySQL 8.0+
- **调度**: PowerJob / 自定义调度
- **规则引擎**: 自定义实现

## 5. 关键实现类详解

### 5.1 QualityCheckEngine (质量检测引擎)

**类位置**: `cn.healthcaredaas.datasphere.svc.quality.engine.QualityCheckEngine`

**职责**: 负责协调规则执行器，执行质量检测的核心引擎。

**核心方法**:

```java
@Component
@RequiredArgsConstructor
@Slf4j
public class QualityCheckEngine {

    private final Map<String, RuleExecutor> executorMap;
    private final JdbcTemplate jdbcTemplate;

    /**
     * 执行质量检测
     *
     * @param rule 质量规则
     * @param dataSource 数据源
     * @return 执行结果
     */
    public RuleExecuteResult executeRule(QualityRule rule, DataSource dataSource) {
        // 1. 获取对应类型的执行器
        RuleExecutor executor = getExecutor(rule.getRuleType());

        // 2. 构建执行上下文
        RuleExecuteContext context = RuleExecuteContext.builder()
            .rule(rule)
            .dataSource(dataSource)
            .build();

        // 3. 执行检测
        return executor.execute(context);
    }

    /**
     * 测试规则有效性
     *
     * @param rule 质量规则
     * @return 是否有效
     */
    public boolean testRule(QualityRule rule) {
        try {
            RuleExecutor executor = getExecutor(rule.getRuleType());
            return executor.validate(rule);
        } catch (Exception e) {
            log.error("规则验证失败: {}", e.getMessage());
            return false;
        }
    }

    private RuleExecutor getExecutor(String ruleType) {
        RuleExecutor executor = executorMap.get(ruleType + "Executor");
        if (executor == null) {
            throw new RuntimeException("不支持的规则类型: " + ruleType);
        }
        return executor;
    }
}
```

### 5.3 CompletenessRuleExecutor (完整性执行器)

**类位置**: `cn.healthcaredaas.datasphere.svc.quality.engine.executor.CompletenessRuleExecutor`

**职责**: 执行数据完整性检查，验证字段是否为空。

```java
@Component
@Slf4j
public class CompletenessRuleExecutor implements RuleExecutor {

    @Override
    public RuleType getSupportedType() {
        return RuleType.COMPLETENESS;
    }

    @Override
    public RuleExecuteResult execute(RuleExecuteContext context) {
        QualityRule rule = context.getRule();
        JdbcTemplate jdbcTemplate = context.getJdbcTemplate();

        // 构建检测SQL: SELECT COUNT(*) FROM table WHERE column IS NULL
        String sql = String.format(
            "SELECT COUNT(*) FROM %s WHERE %s IS NULL",
            rule.getTableName(),
            rule.getColumnName()
        );

        Long errorCount = jdbcTemplate.queryForObject(sql, Long.class);
        Long totalCount = jdbcTemplate.queryForObject(
            "SELECT COUNT(*) FROM " + rule.getTableName(), Long.class
        );

        // 计算得分
        double score = totalCount == 0 ? 100.0 :
            ((totalCount - errorCount) * 100.0 / totalCount);

        return RuleExecuteResult.builder()
            .success(true)
            .totalCount(totalCount)
            .errorCount(errorCount)
            .score(BigDecimal.valueOf(score).setScale(2, RoundingMode.HALF_UP))
            .errorDetails(queryErrorDetails(jdbcTemplate, rule))
            .build();
    }

    @Override
    public boolean validate(QualityRule rule) {
        return StringUtils.isNotBlank(rule.getTableName())
            && StringUtils.isNotBlank(rule.getColumnName());
    }

    private List<ErrorDetail> queryErrorDetails(JdbcTemplate jdbc, QualityRule rule) {
        // 查询具体错误数据
        String sql = String.format(
            "SELECT * FROM %s WHERE %s IS NULL LIMIT 100",
            rule.getTableName(),
            rule.getColumnName()
        );
        // 返回错误详情列表
        return jdbc.query(sql, new ErrorDetailMapper());
    }
}
```

### 5.4 QualityTaskSchedulerService (任务调度服务)

**类位置**: `cn.healthcaredaas.datasphere.svc.quality.service.QualityTaskSchedulerService`

**职责**: 负责检测任务的调度执行和生命周期管理。

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class QualityTaskSchedulerService {

    private final QualityTaskMapper taskMapper;
    private final QualityReportMapper reportMapper;
    private final QualityIssueMapper issueMapper;
    private final QualityCheckEngine checkEngine;
    private final DataSourceService dataSourceService;

    /**
     * 创建检测任务
     *
     * @param ruleId 规则ID
     * @param taskType 任务类型
     * @return 任务ID
     */
    @Transactional
    public String createTask(String ruleId, String taskType) {
        QualityRule rule = ruleMapper.selectById(ruleId);
        if (rule == null) {
            throw new RuntimeException("规则不存在");
        }

        QualityTask task = new QualityTask();
        task.setTaskName("任务_" + rule.getRuleName() + "_" + System.currentTimeMillis());
        task.setRuleId(ruleId);
        task.setDatasourceId(rule.getDatasourceId());
        task.setTableName(rule.getTableName());
        task.setColumnName(rule.getColumnName());
        task.setTaskType(taskType);
        task.setTaskStatus("PENDING");
        task.setScheduleType("NONE");

        taskMapper.insert(task);
        return task.getId();
    }

    /**
     * 执行检测任务
     *
     * @param taskId 任务ID
     * @return 执行结果
     */
    @Transactional
    public QualityReport executeTask(String taskId) {
        QualityTask task = taskMapper.selectById(taskId);
        if (task == null) {
            throw new RuntimeException("任务不存在");
        }

        QualityRule rule = ruleMapper.selectById(task.getRuleId());
        DataSource dataSource = dataSourceService.getDataSource(rule.getDatasourceId());

        // 更新任务状态为运行中
        task.setTaskStatus("RUNNING");
        task.setStartTime(LocalDateTime.now());
        taskMapper.updateById(task);

        try {
            // 执行检测
            RuleExecuteResult result = checkEngine.executeRule(rule, dataSource);

            // 生成报告
            QualityReport report = generateReport(task, rule, result);

            // 记录问题
            if (result.getErrorCount() > 0) {
                saveIssues(task, rule, result.getErrorDetails());
            }

            // 更新任务状态为成功
            task.setTaskStatus("SUCCESS");
            task.setEndTime(LocalDateTime.now());
            taskMapper.updateById(task);

            return report;

        } catch (Exception e) {
            log.error("任务执行失败: {}", e.getMessage());
            task.setTaskStatus("FAIL");
            task.setErrorMessage(e.getMessage());
            task.setEndTime(LocalDateTime.now());
            taskMapper.updateById(task);
            throw e;
        }
    }

    /**
     * 批量执行任务
     */
    @Transactional
    public List<QualityReport> batchExecute(List<String> taskIds) {
        return taskIds.stream()
            .map(this::executeTask)
            .collect(Collectors.toList());
    }

    private QualityReport generateReport(QualityTask task, QualityRule rule,
                                         RuleExecuteResult result) {
        QualityReport report = new QualityReport();
        report.setTaskId(task.getId());
        report.setRuleId(rule.getId());
        report.setReportName("质量报告_" + task.getTaskName());
        report.setReportDate(LocalDateTime.now());
        report.setTotalRows(result.getTotalCount());
        report.setErrorRows(result.getErrorCount());
        report.setScore(result.getScore());
        report.setGrade(calculateGrade(result.getScore()));
        report.setReportContent(generateReportContent(task, rule, result));
        report.setStatus(1);

        reportMapper.insert(report);
        return report;
    }

    private String calculateGrade(BigDecimal score) {
        int value = score.intValue();
        if (value >= 95) return "A";
        if (value >= 85) return "B";
        if (value >= 75) return "C";
        if (value >= 60) return "D";
        return "F";
    }
}
```

```java
        LambdaQueryWrapper<QualityRule> wrapper = new LambdaQueryWrapper<>();

        if (StringUtils.isNotBlank(params.getRuleName())) {
            wrapper.like(QualityRule::getRuleName, params.getRuleName());
        }

        if (StringUtils.isNotBlank(params.getRuleType())) {
            wrapper.eq(QualityRule::getRuleType, params.getRuleType());
        }

        if (params.getStatus() != null) {
            wrapper.eq(QualityRule::getStatus, params.getStatus());
        }

        wrapper.orderByDesc(QualityRule::getCreateTime);

        return baseMapper.selectPage(page, wrapper);
    }

    /**
     * 测试规则有效性
     *
     * @param ruleId 规则ID
     * @return 是否有效
     */
    @Override
    public boolean testRule(String ruleId) {
        QualityRule rule = getById(ruleId);
        if (rule == null) {
            return false;
        }
        return checkEngine.testRule(rule);
    }

    /**
     * 从模板创建规则
     *
     * @param templateId 模板ID
     * @param params 参数
     * @return 规则ID
     */
    @Override
    @Transactional
    public String createRuleFromTemplate(String templateId, Map<String, Object> params) {
        QualityRuleTemplate template = templateMapper.selectById(templateId);
        if (template == null) {
            throw new RuntimeException("模板不存在");
        }

        // 验证参数
        validateTemplateParams(template, params);

        // 替换表达式模板
        String expression = template.getExpressionTemplate();
        for (Map.Entry<String, Object> entry : params.entrySet()) {
            expression = expression.replace("${" + entry.getKey() + "}",
                entry.getValue().toString());
        }

        // 创建规则
        QualityRule rule = new QualityRule();
        rule.setRuleCode("RULE_" + System.currentTimeMillis());
        rule.setRuleName((String) params.getOrDefault("ruleName",
            template.getTemplateName()));
        rule.setRuleType(template.getRuleType());
        rule.setDatasourceId((String) params.get("datasourceId"));
        rule.setTableName((String) params.get("tableName"));
        rule.setColumnName((String) params.get("columnName"));
        rule.setRuleExpression(expression);
        rule.setErrorMessage(template.getErrorMessageTemplate());
        rule.setStatus(1);

        save(rule);
        return rule.getId();
    }

    private void validateTemplateParams(QualityRuleTemplate template,
                                        Map<String, Object> params) {
        if (StringUtils.isBlank(template.getParamDefinition())) {
            return;
        }

        try {
            JSONArray paramDefs = JSON.parseArray(template.getParamDefinition());
            for (int i = 0; i < paramDefs.size(); i++) {
                JSONObject paramDef = paramDefs.getJSONObject(i);
                String paramName = paramDef.getString("name");
                Boolean required = paramDef.getBoolean("required");

                if (Boolean.TRUE.equals(required) && !params.containsKey(paramName)) {
                    throw new RuntimeException("缺少必填参数: " + paramName);
                }
            }
        } catch (Exception e) {
            log.warn("参数验证失败: {}", e.getMessage());
        }
    }
}
```

## 6. 规则引擎设计

### 6.1 支持的规则类型

数据质量模块支持7种核心规则类型，覆盖数据质量管理的各个方面：

| 规则类型 | 英文标识 | 适用场景 | 检测原理 |
|----------|----------|----------|----------|
| **完整性检查** | COMPLETENESS | 字段非空校验 | 检查字段是否为NULL或空字符串 |

### 6.2 规则类型详细说明与配置

#### 6.2.1 完整性检查 (COMPLETENESS)

**规则说明**：
验证字段值是否为NULL或空字符串，确保数据完整性。适用于所有必填字段，如用户姓名、订单编号等。

**配置参数**：

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| table | string | 是 | 表名 | `user_info` |
| column | string | 是 | 字段名 | `phone` |
| emptyCheck | boolean | 否 | 是否检查空字符串 | `true` |

**规则表达式模板**：
```sql
-- MySQL
SELECT * FROM ${table} WHERE ${column} IS NULL
    ${emptyCheck ? "OR " + column + " = ''" : ""}

-- Oracle
SELECT * FROM ${table} WHERE ${column} IS NULL

-- SQLServer
SELECT * FROM ${table} WHERE ${column} IS NULL
```

**创建规则示例**：
```json
{
  "ruleName": "用户手机号完整性检查",
  "ruleType": "COMPLETENESS",
  "datasourceId": "ds_001",
  "tableName": "user_info",
  "columnName": "phone",
  "ruleExpression": "SELECT * FROM user_info WHERE phone IS NULL OR phone = ''",
  "errorMessage": "用户手机号不能为空",
  "status": 1
}
```

**质量评分计算**：
```
score = (总行数 - 空值行数) / 总行数 × 100
```

#### 6.2.2 唯一性检查 (UNIQUENESS)

**规则说明**：
验证字段值在表中的唯一性，适用于主键、业务主键等不能重复的场景。

**配置参数**：

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| table | string | 是 | 表名 | `order_info` |
| columns | string | 是 | 字段名(多个用逗号分隔) | `order_no` 或 `user_id,coupon_code` |

**规则表达式模板**：
```sql

SELECT ${columns}, COUNT(*) as duplicate_count

GROUP BY ${columns}

SELECT t.*
FROM ${table} t
INNER JOIN (

    FROM ${table}
    GROUP BY ${columns}
    HAVING COUNT(*) > 1
) dup ON ${joinCondition}
```

#### 6.2.3 格式检查 (FORMAT)

**规则说明**：
使用正则表达式验证字段值格式，适用于手机号、邮箱、身份证号等有固定格式的数据。

**配置参数**：

#### 6.2.4 值域检查 (VALUE_RANGE)

**规则说明**：
验证字段值是否在指定的数值范围或枚举集合内，适用于年龄、金额、状态码等有限范围的字段。

**配置参数**：

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| table | string | 是 | 表名 | `user_info` |
| column | string | 是 | 字段名 | `age` |
| rangeType | string | 是 | 范围类型 | `BETWEEN`/`IN`/`GREATER_THAN`/`LESS_THAN` |
| min | number | 条件 | 最小值 | `18` |
| max | number | 条件 | 最大值 | `100` |
| values | string | 条件 | 枚举值(逗号分隔) | `0,1,2,3` |
| refTable | string | 条件 | 参考表名 | `dict_status` |
| refColumn | string | 条件 | 参考字段名 | `code` |

**范围类型说明**：

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| BETWEEN | 数值范围 | 年龄、金额、日期范围 |

**规则表达式模板**：

```sql
-- BETWEEN - 数值范围
SELECT * FROM ${table} WHERE ${column} < ${min} OR ${column} > ${max}

-- GREATER_THAN - 大于
SELECT * FROM ${table} WHERE ${column} <= ${threshold}

-- LESS_THAN - 小于
SELECT * FROM ${table} WHERE ${column} >= ${threshold}

-- REFERENCE - 外键引用
SELECT t.* FROM ${table} t
LEFT JOIN ${refTable} r ON t.${column} = r.${refColumn}
WHERE r.${refColumn} IS NULL
```

#### 6.2.5 一致性检查 (CONSISTENCY)

**规则说明**：
验证跨表或跨字段的数据一致性，适用于父子表、关联表之间的数据同步验证。

**支持的多表校验场景**：

| 校验场景 | 说明 | 典型应用 |
|----------|------|----------|
| **外键引用检查** | 检查表A中的字段值在表B中是否存在 | 订单表user_id在用户表中是否存在 |
| **反向存在检查** | 检查表A中存在的数据，表B中不存在 | 用户表中有但员工表中没有的记录 |
| **汇总一致性** | 父表汇总字段与子表明细汇总是否一致 | 订单总额=SUM(订单明细金额) |
| **记录数匹配** | 关联表之间的记录数是否匹配 | 用户订单数统计一致性 |
| **字段值相等** | 关联表中对应字段值是否相等 | 订单金额=支付金额 |
| **时间顺序** | 多个时间字段的逻辑顺序 | 创建<支付<发货<完成 |

**配置参数**：

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| mainTable | string | 是 | 主表名 | `order_info` |
| mainColumn | string | 是 | 主表字段 | `total_amount` |
| refTable | string | 是 | 参考表名 | `order_item` |
| refColumn | string | 是 | 参考表字段 | `item_amount` |
| compareType | string | 是 | 比较类型 | `FOREIGN_KEY`/`SUM_EQUAL`/`COUNT_MATCH`/`REVERSE_EXIST` |
| condition | string | 否 | 关联条件 | `order_id` |

**比较类型说明**：

| 类型 | 说明 | SQL模式 | 示例 |
|------|------|---------|------|
| FOREIGN_KEY | 外键引用检查 | LEFT JOIN + IS NULL | 订单表user_id在用户表中是否存在 |
| REVERSE_EXIST | 反向存在检查 | LEFT JOIN + IS NULL | 用户存在但员工信息不存在 |
| SUM_EQUAL | 汇总值相等 | LEFT JOIN + SUM != | 订单总额 = SUM(子项金额) |
| COUNT_MATCH | 记录数匹配 | LEFT JOIN + COUNT != | 用户订单数统计一致性 |
| FIELD_EQUAL | 字段值相等 | LEFT JOIN + field != | 订单金额 = 支付金额 |
| DATE_SEQUENCE | 日期顺序 | 时间字段比较 | 创建 < 支付 < 发货 |

**规则表达式模板**：

```sql
-- FOREIGN_KEY - 外键引用检查
-- 场景：检查主表字段在参考表中是否存在
-- 原理：LEFT JOIN后，参考表字段为NULL表示不存在
SELECT m.*
FROM ${mainTable} m
LEFT JOIN ${refTable} r ON m.${mainColumn} = r.${refColumn}
WHERE r.${refColumn} IS NULL

-- 或使用 NOT EXISTS（性能更好）
SELECT m.*
FROM ${mainTable} m
WHERE NOT EXISTS (
    SELECT 1 FROM ${refTable} r
    WHERE r.${refColumn} = m.${mainColumn}
)

-- REVERSE_EXIST - 反向存在检查
-- 场景：检查参考表中存在但主表中不存在的数据
-- 示例：用户表中有手机号，但在会员表中不存在
SELECT r.*
FROM ${refTable} r
LEFT JOIN ${mainTable} m ON r.${refColumn} = m.${mainColumn}
WHERE m.${mainColumn} IS NULL

-- SUM_EQUAL - 汇总一致性
SELECT m.*
FROM ${mainTable} m
LEFT JOIN (
    SELECT ${condition}, SUM(${refColumn}) as sum_val
    FROM ${refTable}
    GROUP BY ${condition}
) r ON m.${condition} = r.${condition}
WHERE m.${mainColumn} != r.sum_val
   OR r.sum_val IS NULL

-- COUNT_MATCH - 记录数匹配
SELECT m.*
FROM ${mainTable} m
LEFT JOIN (
    SELECT ${condition}, COUNT(*) as cnt
    FROM ${refTable}
    GROUP BY ${condition}
) r ON m.${condition} = r.${condition}
WHERE m.${mainColumn} != IFNULL(r.cnt, 0)

-- FIELD_EQUAL - 字段值相等
SELECT m.*
FROM ${mainTable} m
LEFT JOIN ${refTable} r ON m.${condition} = r.${condition}
WHERE m.${mainColumn} != r.${refColumn}
   OR r.${refColumn} IS NULL

-- DATE_SEQUENCE - 时间顺序检查
SELECT * FROM ${mainTable}
WHERE ${laterTime} <= ${earlierTime}
   OR (${laterTime} IS NOT NULL AND ${earlierTime} IS NULL)
```

### 6.2.5a 多表存在性校验详解

**场景描述**：
校验表A中的记录在表B中是否存在，这是数据质量检查中最常见的多表校验场景。适用于外键约束、数据同步验证、关联完整性检查等。

**两种校验方向**：

| 方向 | 场景 | SQL模式 | 说明 |
|------|------|---------|------|
| **正向检查** | A表存在但B表不存在 | `A LEFT JOIN B WHERE B.id IS NULL` | 检查A表记录是否都有对应的B表记录 |
| **反向检查** | B表存在但A表不存在 | `B LEFT JOIN A WHERE A.id IS NULL` | 检查B表记录是否都有对应的A表记录 |

**正向检查示例（订单→用户）**：
```sql
-- 检查订单表中的user_id在用户表中是否存在
SELECT o.*
FROM order_info o
LEFT JOIN user_info u ON o.user_id = u.user_id
WHERE u.user_id IS NULL

-- 或使用NOT EXISTS（大数据量时性能更好）
SELECT o.*
FROM order_info o
WHERE NOT EXISTS (
    SELECT 1 FROM user_info u WHERE u.user_id = o.user_id
)
```

**反向检查示例（用户→会员）**：
```sql
-- 检查标记为VIP的用户在VIP表中是否有记录
SELECT u.*
FROM user_info u
LEFT JOIN vip_member v ON u.user_id = v.user_id
WHERE v.user_id IS NULL
  AND u.user_level = 'VIP'
```

**性能优化建议**：

| 数据量 | 推荐方案 | 说明 |
|--------|----------|------|
| < 10万 | LEFT JOIN | 简单直观，便于调试 |
| 10万-100万 | NOT EXISTS | 避免重复扫描，性能更好 |
| > 100万 | 增量检查 | 只检查增量数据，定期全量 |
| 跨库 | 数据同步后检查 | 先同步到同一数据源再检查 |

**一致性检查执行器实现**：

```java
@Component("CONSISTENCYExecutor")
public class ConsistencyRuleExecutor implements RuleExecutor {

    @Override
    public RuleType getSupportedType() {
        return RuleType.CONSISTENCY;
    }

    @Override
    public RuleExecuteResult execute(RuleExecuteContext context) {
        QualityRule rule = context.getRule();
        JdbcTemplate jdbc = context.getJdbcTemplate();

        // 执行多表关联检查SQL
        String checkSql = rule.getRuleExpression();
        List<Map<String, Object>> errorRows = jdbc.queryForList(checkSql);

        long errorCount = errorRows.size();
        long totalCount = getTotalCount(jdbc, rule.getTableName());

        double score = totalCount == 0 ? 100.0 :
            ((totalCount - errorCount) * 100.0 / totalCount);

        return RuleExecuteResult.builder()
            .success(true)
            .totalCount(totalCount)
            .errorCount(errorCount)
            .score(BigDecimal.valueOf(score).setScale(2, RoundingMode.HALF_UP))
            .errorDetails(convertToErrorDetails(errorRows))
            .build();
    }

    @Override
    public boolean validate(QualityRule rule) {
        // 验证SQL必须包含多表关联
        String expr = rule.getRuleExpression().toUpperCase();
        return expr.contains("JOIN") || expr.contains("EXISTS") || expr.contains("IN");
    }

    /**
     * 外键引用检查 - 多表存在性校验核心方法
     * 检查主表字段在参考表中是否存在
     */
    private RuleExecuteResult executeForeignKeyCheck(
            JdbcTemplate jdbc, QualityRule rule) {
        String sql = rule.getRuleExpression();

        // 1. 获取总记录数
        String mainTable = rule.getTableName();
        Long totalCount = jdbc.queryForObject(
            "SELECT COUNT(*) FROM " + mainTable, Long.class);

        // 2. 执行多表关联检查
        List<Map<String, Object>> errorRows = jdbc.queryForList(sql);
        long errorCount = errorRows.size();

        // 3. 计算质量评分
        double score = totalCount == 0 ? 100.0 :
            ((totalCount - errorCount) * 100.0 / totalCount);

        // 4. 构建错误详情
        List<RuleExecuteResult.ErrorDetail> errorDetails = errorRows.stream()
            .map(row -> RuleExecuteResult.ErrorDetail.builder()
                .primaryKey(String.valueOf(row.get("id")))
                .columnName(rule.getColumnName())
                .errorValue(String.valueOf(row.get(rule.getColumnName())))
                .errorMessage("关联数据不存在: " + row.get(rule.getColumnName()))
                .build())
            .limit(100)
            .toList();

        return RuleExecuteResult.builder()
            .success(true)
            .totalCount(totalCount)
            .errorCount(errorCount)
            .score(BigDecimal.valueOf(score).setScale(2, RoundingMode.HALF_UP))
            .errorDetails(errorDetails)
            .message("外键引用一致性检查完成，发现 " + errorCount + " 条异常记录")
            .build();
    }

    /**
     * 反向存在检查
     * 检查参考表中存在但主表中不存在的数据
     */
    private RuleExecuteResult executeReverseExistCheck(
            JdbcTemplate jdbc, QualityRule rule) {
        String sql = rule.getRuleExpression();

        // 执行反向检查
        List<Map<String, Object>> errorRows = jdbc.queryForList(sql);
        long errorCount = errorRows.size();

        // 对于反向检查，总记录数是参考表记录数
        String refTable = extractReferenceTable(sql);
        Long totalCount = jdbc.queryForObject(
            "SELECT COUNT(*) FROM " + refTable, Long.class);

        double score = totalCount == 0 ? 100.0 :
            ((totalCount - errorCount) * 100.0 / totalCount);

        return RuleExecuteResult.builder()
            .success(true)
            .totalCount(totalCount)
            .errorCount(errorCount)
            .score(BigDecimal.valueOf(score).setScale(2, RoundingMode.HALF_UP))
            .errorDetails(convertToErrorDetails(errorRows))
            .message("反向存在检查完成")
            .build();
    }

    /**
     * 从SQL中提取参考表名
     */
    private String extractReferenceTable(String sql) {
        // 简单解析LEFT JOIN后的表名
        String upperSql = sql.toUpperCase();
        int leftJoinIndex = upperSql.indexOf("LEFT JOIN");
        if (leftJoinIndex > 0) {
            String afterJoin = sql.substring(leftJoinIndex + 9).trim();
            return afterJoin.split("\\s+")[0];
        }
        return "";
    }
}
```

**多表存在性校验的核心SQL模式**：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 检查性能差 | 大表JOIN导致全表扫描 | 添加索引，使用EXISTS替代IN |
| 误报过多 | 关联条件不准确 | 细化关联条件，添加业务过滤 |
| 内存溢出 | 返回数据量过大 | 添加LIMIT限制，分页处理 |
| 空值处理 | NULL值导致匹配失败 | 使用IFNULL/COALESCE处理 |

## 9. 扩展指南

### 9.1 自定义规则执行器

参考 `CompletenessRuleExecutor` 实现自定义执行器：

```java
@Component
public class MyCustomRuleExecutor implements RuleExecutor {

    @Override
    public RuleType getSupportedType() {
        return RuleType.CUSTOM; // 或使用自定义类型
    }

    @Override
    public RuleExecuteResult execute(RuleExecuteContext context) {
        QualityRule rule = context.getRule();
        JdbcTemplate jdbc = context.getJdbcTemplate();

        // 1. 构建检测SQL
        String checkSql = buildCheckSql(rule);

        // 2. 执行查询
        List<Map<String, Object>> errors = jdbc.queryForList(checkSql);

        // 3. 计算指标
        long totalCount = getTotalCount(jdbc, rule.getTableName());
        long errorCount = errors.size();
        BigDecimal score = calculateScore(totalCount, errorCount);

        // 4. 构建结果
        return RuleExecuteResult.builder()
            .success(true)
            .totalCount(totalCount)
            .errorCount(errorCount)
            .score(score)
            .errorDetails(convertToErrorDetails(errors))
            .build();
    }

    @Override
    public boolean validate(QualityRule rule) {
        // 验证规则配置
        return StringUtils.isNotBlank(rule.getRuleExpression());
    }
}
```

### 9.2 质量评分算法定制

默认评分算法:
```
score = (total - error) / total * 100
```

自定义权重评分:
```java
public BigDecimal calculateWeightedScore(List<RuleExecuteResult> results) {
    BigDecimal totalScore = BigDecimal.ZERO;
    BigDecimal totalWeight = BigDecimal.ZERO;

    for (RuleExecuteResult result : results) {
        BigDecimal weight = getRuleWeight(result.getRuleId());
        totalScore = totalScore.add(result.getScore().multiply(weight));
        totalWeight = totalWeight.add(weight);
    }

    return totalScore.divide(totalWeight, 2, RoundingMode.HALF_UP);
}
```
