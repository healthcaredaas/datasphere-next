## 一、重构目标与范围

### 1.1 重构目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| **技术升级** | 升级到Spring Boot 3.x + JDK 21 | P0 |
| **架构优化** | 优化模块划分，降低耦合度 | P0 |
| **功能增强** | 新增数据质量、数据资产、数据安全模块 | P1 |
| **多租户移除** | 去除多租户功能，简化架构 | P0 |
| **性能提升** | 利用虚拟线程等新特性提升性能 | P1 |

### 1.2 重构范围

**包含内容**:
- ✅ 所有现有服务模块升级
- ✅ 新增数据质量服务
- ✅ 新增数据资产服务
- ✅ 新增数据安全服务
- ✅ 数据库脚本重构
- ✅ 配置外部化改造

**不包含内容**:
- ❌ 前端页面改造（后续迭代）
- ❌ 已有业务逻辑大幅修改
- ❌ HIE增强功能（第二阶段）

### 1.3 重构原则

1. **最小改动原则**: 仅修改必要代码，保持原有功能不变
2. **渐进式重构**: 分模块逐步迁移，降低风险
3. **兼容过渡**: 提供兼容层，支持平滑过渡
4. **充分测试**: 每个模块完成后进行充分测试

## 三、多租户移除方案

### 3.1 多租户功能现状

**当前实现**:
- 所有实体类包含 `tenant_id` 字段
- MyBatis拦截器自动注入租户过滤条件
- 服务层通过TenantContext获取当前租户

**影响范围**:
- 40+张数据库表
- 60+个实体类
- 20+个Service类
- MyBatis配置

### 3.2 移除步骤

#### 步骤1: 移除依赖

```xml
<!-- 从pom.xml中移除 -->
<dependency>
    <groupId>cn.healthcaredaas</groupId>
    <artifactId>daas-tenant-core</artifactId>
</dependency>
<dependency>
    <groupId>cn.healthcaredaas</groupId>
    <artifactId>daas-tenant-starter</artifactId>
</dependency>
```

#### 步骤2: 移除实体类tenant_id字段

**修改前**:
```java
@Data
public class DatasourceInfo {
    private String id;
    private String tenantId;  // 需要移除
    private String datasourceCode;
    private String datasourceName;
    // ...
}
```

**修改后**:
```java
@Data
public class DatasourceInfo {
    private String id;
    private String datasourceCode;
    private String datasourceName;
    // ...
}
```

#### 步骤3: 移除MyBatis拦截器配置

```yaml
# application.yml 修改前
mybatis-plus:
  configuration:
    interceptors:
      - cn.healthcaredaas.daas.tenant.interceptor.TenantLineInterceptor

# application.yml 修改后
mybatis-plus:
  configuration:
    # 移除租户拦截器配置
```

#### 步骤4: 移除Service层租户代码

```java
// 修改前
@Service
public class DatasourceServiceImpl implements DatasourceService {

    @Override
    public List<DatasourceInfo> list() {
        String tenantId = TenantContext.getTenantId();
        return datasourceDao.selectList(
            new QueryWrapper<DatasourceInfo>()
                .eq("tenant_id", tenantId)
        );
    }
}

// 修改后
@Service
public class DatasourceServiceImpl implements DatasourceService {

    @Override
    public List<DatasourceInfo> list() {
        return datasourceDao.selectList(null);
    }
}
```

#### 步骤5: 数据库脚本清理

```sql
-- V1.0.1__remove_tenant.sql
-- 移除所有表的tenant_id字段

ALTER TABLE datasource_info DROP COLUMN tenant_id;
ALTER TABLE datasource_type DROP COLUMN tenant_id;
ALTER TABLE datasource_classify DROP COLUMN tenant_id;
-- ... 其他表

-- 移除租户相关索引
DROP INDEX IDX_datasource_info_tenant ON datasource_info;
-- ... 其他索引
```

### 3.3 数据隔离替代方案

如果业务需要数据隔离，提供以下替代方案:

#### 方案A: 应用级别隔离

通过 `create_by` + 数据权限实现隔离:

```java
@Service
public class DatasourceServiceImpl implements DatasourceService {

    @Autowired
    private DataPermissionService permissionService;

    @Override
    public List<DatasourceInfo> list() {
        // 根据当前用户权限过滤
        Set<String> allowedIds = permissionService.getAllowedDatasourceIds();
        if (allowedIds.isEmpty()) {
            return Collections.emptyList();
        }
        return datasourceDao.selectList(
            new QueryWrapper<DatasourceInfo>()
                .in("id", allowedIds)
        );
    }
}
```

#### 方案B: 项目级别隔离

通过项目归属实现隔离:

```java
// 在实体中增加project_id字段
@Data
public class DatasourceInfo {
    private String id;
    private String projectId;  // 所属项目
    // ...
}
```

### 4.3 Jakarta迁移脚本

```bash
#!/bin/bash
# migrate-to-jakarta.sh

# 批量替换javax为jakarta
find . -name "*.java" -type f -exec sed -i '' 's/import javax\.servlet/import jakarta.servlet/g' {} \;
find . -name "*.java" -type f -exec sed -i '' 's/import javax\.validation/import jakarta.validation/g' {} \;
find . -name "*.java" -type f -exec sed -i '' 's/import javax\.annotation/import jakarta.annotation/g' {} \;
find . -name "*.java" -type f -exec sed -i '' 's/import javax\.persistence/import jakarta.persistence/g' {} \;
find . -name "*.java" -type f -exec sed -i '' 's/import javax\.ws\.rs/import jakarta.ws.rs/g' {} \;

echo "Jakarta迁移完成，请检查并手动修复遗留问题"
```
