## 一、环境要求

### 1.1 硬件要求

| 环境 | CPU | 内存 | 磁盘 | 网络 |
|------|-----|------|------|------|
| 开发环境 | 4核+ | 8GB+ | 100GB+ | 内网 |
| 测试环境 | 8核+ | 16GB+ | 500GB+ | 内网 |
| 生产环境 | 16核+ | 32GB+ | 1TB+ SSD | 专线 |

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 21+ | OpenJDK或Oracle JDK |

| Redis | 7.0+ | 缓存 |
| RocketMQ | 5.3+ | 消息队列 |

## 四、单机部署

### 4.1 编译打包

```bash
# 克隆代码
git clone https://github.com/healthcaredaas/datasphere-next.git
cd datasphere-next

# 编译
mvn clean package -DskipTests

# 查看打包结果
ls -la */target/*.jar
```

### 4.2 启动服务

**启动脚本: start.sh**

```bash
#!/bin/bash

APP_NAME=$1
JAR_FILE="${APP_NAME}/target/${APP_NAME}-2.0.0-SNAPSHOT.jar"
LOG_FILE="logs/${APP_NAME}.log"

# 创建日志目录
mkdir -p logs

# JVM参数
JAVA_OPTS="-server -Xms1g -Xmx1g -Xmn512m
  -XX:MetaspaceSize=128m
  -XX:MaxMetaspaceSize=256m
  -XX:+UseG1GC
  -XX:MaxGCPauseMillis=200
  -XX:+HeapDumpOnOutOfMemoryError
  -XX:HeapDumpPath=logs/heapdump.hprof
  -Dfile.encoding=UTF-8
  -Duser.timezone=Asia/Shanghai"

# 启动
nohup java ${JAVA_OPTS} -jar ${JAR_FILE} > ${LOG_FILE} 2>&1 &

echo "${APP_NAME} started, PID: $!"
```

**启动顺序**

```bash
# 1. 启动核心服务
./start.sh datasphere-svc-datasource

# 2. 等待10秒
sleep 10

# 3. 启动数据集成服务
./start.sh datasphere-svc-integration

# 4. 等待10秒
sleep 10

# 5. 启动其他服务
./start.sh datasphere-svc-standard
./start.sh datasphere-svc-master
./start.sh datasphere-svc-metadata
./start.sh datasphere-svc-quality
./start.sh datasphere-svc-asset
./start.sh datasphere-svc-security
```

### 4.3 停止服务

```bash
#!/bin/bash
# stop.sh

APP_NAME=$1

# 查找进程
PID=$(ps -ef | grep "${APP_NAME}" | grep -v grep | awk '{print $2}')

if [ -n "$PID" ]; then
    echo "Stopping ${APP_NAME}, PID: ${PID}"
    kill -15 ${PID}

    # 等待进程结束
    for i in {1..30}; do
        if ! ps -p ${PID} > /dev/null; then
            echo "${APP_NAME} stopped"
            exit 0
        fi
        sleep 1
    done

    # 强制结束
    echo "Force stopping ${APP_NAME}"
    kill -9 ${PID}
else
    echo "${APP_NAME} is not running"
fi
```

## 五、Docker部署

### 5.1 Dockerfile

```dockerfile
# datasphere-svc-datasource/Dockerfile

FROM eclipse-temurin:21-jre-alpine

LABEL maintainer="chirspan <chenpan.ai@qq.com>"

# 设置时区
ENV TZ=Asia/Shanghai
RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone

# 创建工作目录
WORKDIR /app

# 复制jar包
COPY target/datasphere-svc-datasource-2.0.0-SNAPSHOT.jar app.jar

# JVM参数
ENV JAVA_OPTS="-Xms1g -Xmx1g -XX:+UseG1GC"

# 暴露端口
EXPOSE 8081

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=60s --retries=3 \
  CMD wget --quiet --tries=1 --spider http://localhost:8081/actuator/health || exit 1

# 启动命令
ENTRYPOINT exec java ${JAVA_OPTS} -jar app.jar
```

**docker-compose.yml**


## 八、常见问题

### 8.1 启动失败

**问题**: 服务启动报错 `Connection refused` to Nacos

**解决**:
```bash
# 1. 检查Nacos是否启动
curl http://localhost:8848/nacos/v1/console/health/readiness

# 2. 检查网络连通性
telnet localhost 8848

# 3. 检查配置是否正确
vim bootstrap.yml
```

### 8.2 数据库连接失败

**问题**: 报错 `Communications link failure`

**解决**:
```bash
# 1. 检查MySQL是否启动
systemctl status mysqld

# 2. 检查连接信息
mysql -u datasphere -p -h localhost datasphere

# 3. 检查防火墙
firewall-cmd --list-ports
```

### 8.3 内存不足

**问题**: 报错 `OutOfMemoryError`

**解决**:
```bash
# 1. 调整JVM参数
JAVA_OPTS="-Xms2g -Xmx2g"

# 2. 检查内存使用
jmap -heap <pid>

# 3. 分析堆内存
jmap -dump:format=b,file=heap.hprof <pid>
```

### 8.4 端口冲突

**问题**: 报错 `Port already in use`

**解决**:
```bash
# 1. 查找占用端口的进程
netstat -tlnp | grep 8081

# 2. 结束进程
kill -9 <pid>

# 3. 或修改服务端口
vim application.yml
# server.port=8082
```

### 8.5 时区问题

**问题**: 数据库时间与系统时间不一致

**解决**:
```bash
# 1. 检查时区
date
timedatectl

# 2. 设置时区
timedatectl set-timezone Asia/Shanghai

# 3. MySQL时区设置
mysql -e "SET GLOBAL time_zone = '+8:00';"
```
