## 五、关键优化点说明

### 5.1 移除 daas-tenant (多租户模块)

**优化前:**
```
daas-tenant/
├── daas-tenant-core/          - 租户上下文、基础实体
├── daas-tenant-mybatis/       - 动态数据源管理
├── daas-tenant-redis/         - 动态Redis管理
└── daas-tenant-sdk-all/       - SDK聚合包
```

**优化后:**
```
【完全移除】
- 减少4个模块
- 减少约4000行代码
- 简化数据源配置
- 简化Redis配置
```

### 5.2 合并 web-protect 到 web-core

**优化前:**
```
daas-web/
├── daas-web-core/             - Web基础
└── daas-web-protect/          - 安全防护 (单独模块)
```

**优化后:**
```
daas-web/
└── daas-web-core/             - Web基础 + 安全防护
```

**理由:**
- web-protect 功能单一，仅包含加密工具类
- 与 web-core 职责高度相关
- 合并后减少模块间依赖

### 5.3 合并 web-rest-tenant 功能到 web-rest

**优化前:**
```
daas-web/
├── daas-web-rest/             - 基础REST控制器
└── daas-web-rest-tenant/      - 多租户REST控制器
```

**优化后:**
```
daas-web/
└── daas-web-rest/             - 通用REST控制器 (去除租户特性)
```

**具体变化:**
- `TenantBaseCRUDController.java` → `BaseCRUDController.java`
- `TenantBaseVo.java` → `BaseVo.java`
- 移除 `@TenantRequestParam` 注解
- 移除租户参数切面

### 5.4 实体类继承优化

**优化前:**
```java
// 多租户实体
public class RbacUser extends TenantEntity {
    private String tenantId;    // 自动注入
    private String username;
    // ...
}
```

**优化后:**
```java
// 普通实体
public class RbacUser extends BaseEntity {
    private String username;    // 无租户字段
    // ...
}
```

**影响范围:**
- 所有业务实体类 (~20个)
- 数据库表结构 (移除tenant_id字段)
- DAO层查询 (移除tenant_id条件)

## 七、优化收益

### 7.1 架构层面

| 收益项 | 说明 |
|--------|------|
| **复杂度降低** | 减少9个模块，依赖关系更清晰 |
| **维护成本降低** | 代码量减少7000行，bug概率降低 |
| **启动速度提升** | 移除动态数据源初始化，启动更快 |
| **配置简化** | 无需配置多租户数据源和Redis |

### 7.2 功能层面

| 收益项 | 说明 |
|--------|------|
| **性能提升** | 无租户拦截器，请求处理更快 |
| **内存节省** | 无动态数据源连接池缓存 |
| **调试简化** | 单数据源，问题定位更容易 |

### 7.3 开发层面

| 收益项 | 说明 |
|--------|------|
| **学习成本降低** | 无需理解多租户架构 |
| **开发效率提升** | 实体类简化，无需关注tenant_id |
| **测试简化** | 单租户场景，测试用例更简洁 |
